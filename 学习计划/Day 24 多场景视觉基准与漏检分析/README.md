# Day 24：多场景视觉基准与漏检分析

## 本日目标

Day 23 证明视觉观察 API 能从一张私有截图返回守卫验证后的只读观察，但单张截图只能发现一次性的遗漏，不能说明模型在不同场景中的表现。

Day 24 不训练模型，也不扩大机器人权限。它建立一套多场景私有基准和分析脚本，回答三个更具体的问题：

1. 视觉链路在不同场景下是否都能返回结构合格的观察？
2. 场景、危险、方块和实体四类标签中，哪一类最容易漏掉？
3. 漏检是否集中在距离、光照、室内外或画面密度等条件？

## 建议采集的六类截图

每张图都由玩家主动截取，并在本地人工填写“确定看见的必需标签”。不要把猜测、遮挡严重或只能从游戏状态推断出的内容写进标注。为与视觉服务的输出上限保持一致，每张图最多填写 6 个必需方块和 4 个必需实体；超过上限的内容应选出最能代表该场景的项目，而不是全部罗列。

| 建议 ID | 场景 | 主要观察点 |
| --- | --- | --- |
| `daylight_open_near` | 白天、开阔、近景 | 基础方块与场景标签 |
| `daylight_open_far` | 白天、开阔、远景 | 远处方块和结构漏检 |
| `night_or_low_light` | 夜晚或低光照 | 光照变化下的稳定性 |
| `indoor_dense` | 室内或高方块密度 | 材质细节和遮挡 |
| `water_or_height_hazard` | 水边、高处或其他可见危险 | 危险标签是否保守 |
| `entity_present` | 有清晰可见实体 | 实体标签与方块标签的区分 |

六类是起点，不是硬性上限。某一类别可以收集多张图片，但应避免同一画面只改变一点视角后重复计数。

## 运行批量评测

先确保 Day 23 本地隧道仍可访问 `http://127.0.0.1:18768`，再在仓库根目录执行：

```powershell
python "学习计划\Day 23 视觉观察API与基准评测\run_vision_benchmark.py" `
  --manifest "学习计划\Day 24 多场景视觉基准与漏检分析\vision_benchmark_manifest.json" `
  --gateway-url http://127.0.0.1:18768 `
  --report "学习计划\Day 24 多场景视觉基准与漏检分析\reports\vision_benchmark_report.json"
```

该命令会把每张图经 Day 23 的只读 API 发送到视觉模型。报告中的“必需标签覆盖率”是人工确认标签中被模型命中的比例，**不是整体准确率**，也不会惩罚模型额外给出的未经计分标签。

## 运行漏检分析

```powershell
python "学习计划\Day 24 多场景视觉基准与漏检分析\analyze_vision_benchmark.py" `
  --report "学习计划\Day 24 多场景视觉基准与漏检分析\reports\vision_benchmark_report.json" `
  --metadata "学习计划\Day 24 多场景视觉基准与漏检分析\vision_case_metadata.json" `
  --output "学习计划\Day 24 多场景视觉基准与漏检分析\reports\vision_error_analysis.json"
```

输出会包含：守卫接受率、四类标签各自的平均覆盖率、最常漏掉的人工标签，以及按场景元数据标签分组的覆盖率。请优先看重复出现的漏检，而不是根据单个案例仓促修改提示词或更换模型。

私有评测报告还会保存每张图被守卫接受的结构化观察对象，不含图片字节或原始模型文本。它可用于检查“模型看见了同一物体但命名不同”和“模型完全没有提到该物体”之间的区别。

若报告显示某些案例未被守卫接受，先记录其 `reason`。只有需要调查模型输出与固定契约为何不兼容时，才按 Day 23 README 临时启用远端终端诊断；不要为了提高接受率而直接放宽场景或危险标签白名单。

## 解释边界

- 模型返回的观察仍是只读线索，不能直接驱动 Mindcraft 命令。
- 人工标签只记录确定可见的内容；未标注不等于画面中不存在。
- 只有案例数逐步增加、分组之间有对比时，才能提出关于视觉能力的有限结论。
- 所有真实游戏截图、标注和报告保持本地私有，不进入 Git 仓库。

## 首轮多场景基准（2026-07-24）

在六张私有人工标注截图上完成了首轮评测。结果如下：

- 观察守卫接受率：`6/6`，即六次 API 调用均返回结构合格的只读观察。
- 严格必需标签平均覆盖率：`30.66%`。
- 场景标签平均覆盖率：`68.75%`。
- 方块标签平均覆盖率：`26.11%`。
- 危险标签平均覆盖率：`16.67%`。
- 实体标签平均覆盖率：`0%`。

首轮过程中发现并修复了两项评测基础设施问题：`desert` 是稳定出现且安全的只读场景标签，已被加入受控词表；人工方块标签数量曾超过模型最多 6 项的输出上限，现由清单校验强制限制。修正后得到的 `30.66%` 才是容量一致条件下的严格基线。

这 6 张图不能代表模型整体能力，且各细分场景大多只有 1 个样本。它们已经足以指出下一步重点：补充实体和室内高密度场景，区分“同物不同命名”和“完全漏检”，并持续保留精确标签覆盖率而不将其宣传为通用准确率。
